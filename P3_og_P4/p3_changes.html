Endringer

Ettersom vi brukte mye tid på design av nettsiden i foregående innlevering (P2) har vi ikke endret mye fra mockupsene og de orginale tankene vedrørende design. Stilen og uttrykket er likt. Endringene som er gjort er først og fremst små endringer for å gi kontinuitet på nettsiden samt små estetiske justeringer.

Noen spesifikke endringer:

Homepage.html

Endret uttrykk på tjenesterikonene samt tilførte Javascript for hoverfunksjon på disse ikonene for å tydligjøre muligheten til å navigere gjennom disse på hovedsiden.

referanser.html

Små designvalg og endringer av layout for kontinuitet. Endret slideshow til å komme på referansesiden istedenfor hovedsiden for å gi den litt mer action.

grafisk_profil.html

Grafisk profil fikk en "call to action" seksjon over footer slik at bruker enkelt kan stille spørsmål og kontakte oss til enhver tid. Dette gjorde vi også for å gi kontinuitet ettersom dette er en repetert seksjon på nesten hver side.

kontakt_oss.html

På kontakt_oss.html ville vi ha et script som gjør det mulig å sende mail til oss dersom kunden hadde spørsmål. Ettersom dette ikke er mulig i Javascript endret vi dette til å bli validation.js som minner bruker på å fylle inn alle felter før hun/han sender inn forespørselen.

seo.html

Små justeringer av pakker.

nettsidedesign.html

Sikkerhetsseksjonen fikk egne responsive ikoner og erstattet placeholder.

Modularitet

Som nevnt i P2 har vi forsøkt å eliminere repetert kode gjennom flere tiltak. Vi bruker én CSS med seksjoner for egenskaper som går igjen. Herunder; struktur, layout, hovedbilde, fontstørrelser, farger, osv. Navigeringsbar og footer som repeteres er hentet inn med referanse til script i html.

Responsivitet

Ettersom vi ikke visste hvordan vi skulle oppnå responsivitet på nettsiden i P2, kommenterer vi det her. Nettsiden er ikke optimalisert for bruk på mobil men er responsiv nok til å fungere her også. Bruk av FlexBox og @media gjør nettsiden responsiv og estetisk tilfredstillende uansett hvor ubehagelig rare vindusdimensjoner du velger å bruke.

Oppdatert mappestruktur

Endringen i mappestrukturen er gjort som følge av prosjektets gang og økt behov for struktur (egne mapper for bilder, nye navn osv.).

Utover hovedansvar

Alle hadde ansvar for hver sin html og CSS for denne/disse html og minst én Javascriptfunskjon (se tabell over). Dette er ekskludert Mats som hadde hovedansvar for Javascript og CSS til navbar og canvasballene på hovedside.html.
Utover hovedansvaret har Ingeborg, Jakob og Mathias mot slutten sørget for ferdigstilling og justering av CSS og html til å bli responsiv og fungerende som helhet.

Mer spesifikt har Ingeborg fikset CSS og html for hovedbilde og tekst på alle sider slik at bilde fyller hele vinduet (og bare vinduet). Hun har også gjort alle de blå og hvite faktaboksene responsive til vindusendringer samt fikset css for disse. Pakkene under seo.html er også justert og fikset i CSS og html. Hun har også gjort Logo / Sosiale Medier / epost responsive.

Jakob har kodet slideshowet på referanser.html og har gjort tjenester på hovedside.html, pakker på seo.html og person-boksene på kontakt_oss.html responsive.

Mats har utover original plan for navbar gjort denne responsiv og effektiv for alle vindusstørrelser. Har også kodet Javascript for de flyvende ballene på hovedside.html.

Mathias har rettet opp nettsidedesign.html, grafiskprofil.html og seo.html slik at de tilfredsstiller W3C markup validation. Han har også designet favicon, buttons (tjenester på hovedside.html og sikkerhet på nettsidedesign.html) samt CSS og html for "sikkerhet"-seksjonen på nettsidedesign.html. Ellers har han holdt struktur i filer samt laget filstruktur for ny.css slik at vi holder oss til én CSS.

Edvard var tidlig ute med å gjøre CSS og html for nettsidedesign.html, grafisk_profil.html og seo.html, der CSS ble mye brukt på andre sider. Disse er endret endel siden opprinnelig versjon.
